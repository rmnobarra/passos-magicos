Fase 0
Setup do projeto
1
Criar a estrutura de diretórios
Crie a pasta do projeto e copie os arquivos gerados (CLAUDE.md, agents/, .claude/) para dentro dela.
bash
mkdir passos-magicos && cd passos-magicos

# Copie os arquivos gerados pelo Claude para esta pasta:
# CLAUDE.md, agents/, .claude/

mkdir -p app/model src tests monitoring \
  data/raw data/processed notebooks \
  logs reports/figures

git init
git add CLAUDE.md agents/ .claude/
git commit -m "chore: estrutura inicial e instruções Claude Code"
2
Baixar o dataset
Faça download do dataset no Kaggle e coloque o CSV na pasta correta.
bash
# Acesse: https://www.kaggle.com/datasets/filipiimperial/passos-magicos
# Baixe o arquivo e mova para:
mv ~/Downloads/PEDE_PASSOS_DATASET_FIAP.csv data/raw/
⚠️ Se não tiver acesso ao dataset ainda, não se preocupe — o Claude Code vai gerar dados sintéticos automaticamente para que você possa desenvolver e testar o projeto.
3
Criar ambiente virtual e iniciar o Claude Code
bash
python -m venv .venv
source .venv/bin/activate  # Linux/Mac
# .venv\Scripts\activate   # Windows

# Iniciar o Claude Code na raiz do projeto
claude
✅ O Claude Code detecta o CLAUDE.md automaticamente e carrega todo o contexto do projeto. Você verá a confirmação no terminal.
Fase 1
Infraestrutura base
4
Gerar requirements.txt, Dockerfile e docker-compose
Cole este prompt no Claude Code:
prompt → claude code
> Leia o arquivo agents/devops_engineer.md e crie todos os arquivos sob responsabilidade desse agente: requirements.txt, Dockerfile, docker-compose.yml, .dockerignore, .gitignore e .env.example
💡 Após a criação, confira se o requirements.txt foi gerado corretamente antes de seguir.
5
Instalar dependências
bash
pip install -r requirements.txt
Fase 2
Pipeline de Machine Learning
6
Criar os módulos de dados
Preprocessing, feature engineering e utilitários.
prompt → claude code
> Leia o arquivo agents/data_engineer.md e crie os arquivos src/utils.py, src/preprocessing.py e src/feature_engineering.py exatamente conforme as especificações. Inclua type hints e docstrings em português em todas as funções.
7
Criar pipeline de treinamento e avaliação
prompt → claude code
> Leia o arquivo agents/ml_engineer.md e crie src/train.py e src/evaluate.py. Certifique-se de que o script src/train.py possa ser executado diretamente via linha de comando com argumento --data para o caminho do CSV.
8
Treinar o modelo
Execute o treinamento — o Claude Code usará dados sintéticos se o CSV não estiver disponível.
bash · ou use o comando /train
# Com o dataset real:
python src/train.py --data data/raw/PEDE_PASSOS_DATASET_FIAP.csv

# Ou peça ao Claude Code para gerar dados e treinar:
prompt → claude code (alternativo)
> Gere um dataset sintético com 500 registros no schema Passos Mágicos em data/raw/synthetic_data.csv e execute o treinamento. Salve o modelo em app/model/model.joblib e mostre as métricas.
✅ Resultado esperado: F1-Score Macro ≥ 0.70 e arquivo app/model/model.joblib criado.
Fase 3
API FastAPI
9
Criar a API completa
prompt → claude code
> Leia o arquivo agents/api_engineer.md e crie app/schemas.py, app/routes.py, app/middleware.py e app/main.py. A API deve ter os endpoints /predict, /health e /metrics com documentação Swagger automática.
10
Testar a API localmente
bash
# Terminal 1 — subir a API
uvicorn app.main:app --reload --port 8000

# Terminal 2 — testar health
curl -s http://localhost:8000/api/v1/health | python -m json.tool

# Terminal 2 — testar predição
curl -X POST http://localhost:8000/api/v1/predict \
  -H "Content-Type: application/json" \
  -d '{"inde":4.2,"ian":3.8,"ida":4.5,"ieg":5.0,
      "iaa":6.0,"ips":5.5,"ipp":5.2,"ipv":4.8,
      "fase":2,"ano":2024,"ponto_de_virada":false}'
🌐 Swagger UI disponível em http://localhost:8000/docs para testar visualmente todos os endpoints.
Fase 4
Testes unitários
11
Gerar todos os testes unitários
prompt → claude code
> Leia o arquivo agents/qa_engineer.md e crie tests/conftest.py, tests/test_preprocessing.py, tests/test_feature_engineering.py, tests/test_train.py, tests/test_evaluate.py e tests/test_api.py. Use fixtures do conftest para dados sintéticos e garanta cobertura ≥ 80%.
12
Executar os testes e verificar cobertura
bash · ou use o comando /test
pytest tests/ \
  --cov=src \
  --cov=app \
  --cov-report=term-missing \
  --cov-fail-under=80 \
  -v
⚠️ Se a cobertura ficar abaixo de 80%, peça ao Claude Code: "Cobertura atual é X%. Adicione testes para aumentar a cobertura do módulo src/preprocessing.py"
Fase 5
Monitoramento e drift
13
Criar o detector de drift e dashboard
prompt → claude code
> Leia o arquivo agents/qa_engineer.md e crie monitoring/drift_detector.py e monitoring/dashboard.py conforme especificado. O dashboard deve exibir KPIs de predições e permitir baixar o relatório de drift mais recente.
Fase 6
Deploy com Docker
14
Build e deploy completo
bash · ou use o comando /deploy
# Subir API + Monitor com Docker Compose
docker-compose up --build -d

# Verificar containers rodando
docker-compose ps

# Ver logs em tempo real
docker-compose logs -f api
🚀 API em http://localhost:8000/docs · Dashboard em http://localhost:8501
Fase 7
CI/CD com GitHub Actions
15
Criar repositório e fazer o primeiro push
O GitHub Actions só funciona após o código estar no repositório remoto.
bash
# Criar repositório no GitHub (via gh CLI ou interface web)
gh repo create passos-magicos --private --source=. --push

# Ou manualmente:
git remote add origin https://github.com/SEU_USER/passos-magicos.git
git add . && git commit -m "feat: projeto completo MLOps Passos Mágicos"
git push -u origin main
16
Gerar os workflows de CI e CD
prompt → claude code
> Leia o arquivo agents/github_actions_engineer.md e crie os três workflows: .github/workflows/ci.yml (lint + test + docker build), .github/workflows/cd.yml (deploy na Render via API) e .github/workflows/train.yml (retreinamento agendado). Crie também o .github/pull_request_template.md.
✅ O CI dispara em todo push e PR. O CD dispara apenas em push para main — após o CI passar.
17
Configurar os secrets no GitHub
Acesse Settings → Secrets and variables → Actions no seu repositório e adicione:
github secrets
RENDER_API_KEY       ← Render → Account Settings → API Keys
RENDER_SERVICE_ID    ← URL do serviço: srv-XXXXXXXXXXXX
RENDER_SERVICE_URL   ← https://passos-magicos-api.onrender.com
⚠️ Os secrets só podem ser configurados depois que o serviço na Render for criado (próxima fase). Volte aqui após a Fase 8.
Fase 8
Deploy na Render
18
Gerar render.yaml e startup script
prompt → claude code
> Leia o arquivo agents/render_engineer.md e crie o render.yaml na raiz do projeto e o app/render_startup.sh. Atualize o Dockerfile para usar o startup script como CMD. O script deve treinar o modelo automaticamente se o disco persistente estiver vazio.
💡 O render.yaml define toda a infraestrutura como código: serviço, disco persistente de 1 GB para o modelo, health check e auto-deploy.
19
Criar o serviço na Render e validar o deploy
Com o render.yaml no repositório, crie o serviço via Blueprint:
render · passo a passo
1. Acesse https://render.com → New + → Blueprint
2. Conecte o repositório GitHub passos-magicos
3. Render lê o render.yaml e cria o serviço automaticamente
4. Aguarde o primeiro build (≈ 5–10 min, inclui treino do modelo)
5. Copie a URL pública e o Service ID para os secrets do GitHub
bash · validar API em produção
SERVICE_URL="https://passos-magicos-api.onrender.com"

# Health check
curl -s "$SERVICE_URL/api/v1/health" | python -m json.tool

# Predição de alto risco
curl -X POST "$SERVICE_URL/api/v1/predict" \
  -H "Content-Type: application/json" \
  -d '{"inde":3.8,"ian":3.5,"ida":4.0,"ieg":4.5,
      "iaa":5.0,"ips":4.2,"ipp":4.8,"ipv":3.9,
      "fase":2,"ano":2024,"ponto_de_virada":false}'
🚀 Swagger público em https://passos-magicos-api.onrender.com/docs — inclua esse link no README e no vídeo do datathon.
Fase 9
Documentação final
20
Gerar o README.md completo
prompt → claude code
> Crie um README.md completo para o projeto Passos Mágicos seguindo os requisitos do CLAUDE.md. Inclua: visão geral, estrutura de diretórios, instruções de deploy (local, Docker e Render), exemplos de chamadas cURL, explicação da pipeline de ML, justificativa das métricas e escolha do modelo, badges de Python, Docker, pytest e status do CI/CD.
Fase 10
Roteiro da apresentação
21
Gerar o roteiro do vídeo gerencial
O datathon exige um vídeo de até 5 minutos com pelo menos um integrante do grupo, em formato gerencial. O agente roteirista gera o roteiro completo com falas, tempos e dicas de tela.
prompt → claude code
> Leia o arquivo agents/roteirista.md e crie docs/roteiro_apresentacao.md com o roteiro completo de 5 minutos para a apresentação gerencial. Carregue as métricas reais do modelo em app/model/metadata.joblib para preencher os valores de F1 e ROC-AUC no roteiro.
🎬 O roteiro tem 5 blocos cronometrados: problema → solução → demo ao vivo → métricas → impacto. Use o comando /roteiro para gerar diretamente.
22
O que preparar antes de gravar
checklist de gravação
✅ API rodando na Render com URL pública acessível
✅ Swagger UI aberto em /docs pronto para demo ao vivo
✅ GitHub Actions com pipeline verde visível
✅ Dashboard Streamlit com métricas carregadas
✅ Gráfico de feature importance salvo em reports/figures/
✅ Roteiro impresso ou em segundo monitor para consulta
✅ Resolução mínima 1080p · microfone externo · fundo neutro
⏱️ Ensaie o roteiro em voz alta antes de gravar. A 125 palavras/minuto, 5 minutos = ~625 palavras. Não passe disso — a banca corta no tempo.
